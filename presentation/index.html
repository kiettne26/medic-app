07 — TÍNH NĂNG

Cơ chế Pessimistic Lock — Chống đặt trùng

Vấn đề: Race Condition

Khi 2 bệnh nhân cùng bấm đặt một khung giờ gần như đồng thời, nếu không xử lý → cả 2 đều đặt thành công → xung đột dữ liệu.

Giải pháp: Pessimistic Write Lock

// Repository - khoá hàng cơ sở dữ liệu @Lock(LockModeType.PESSIMISTIC_WRITE) @Query("SELECT t FROM TimeSlot t WHERE t.id = :id") Optional<TimeSlot> findByIdWithLock(UUID id); // Service - Transaction Serializable @Transactional(isolation = Isolation.SERIALIZABLE) public Booking createBooking(...) { TimeSlot slot = repo.findByIdWithLock(id); // Block mọi request khác... }

Cơ chế hoạt động

👤 A
Request #1 — Đặt thành công ✅
Được lock trước → Kiểm tra slot trống → Đặt lịch → Commit → Release lock
⬇️ Lock được giải phóng
👤 B
Request #2 — Bị từ chối ❌
Phải chờ lock → Khi vào thấy slot đã booked → Trả lỗi "Khung giờ đã được đặt"
🔒 Bảo vệ 2 lớp
Pessimistic Lock (DB row-level) + Optimistic Lock (@Version) → Đảm bảo 100% không xung đột.

11 — TÍNH NĂNG

Đặt lịch thông minh & Lọc giờ quá khứ

Tính năng nổi bật

  • Hiển thị ca khám khả dụng trong 7 ngày tới
  • Tự động lọc khung giờ đã qua nếu chọn "Hôm nay"
  • Giao diện chọn ngày cuộn ngang trực quan
  • Xác nhận gửi Email + QR Code tức thì
⏰ Bộ lọc thời gian thực
So sánh TimeOfDay.now() với giờ bắt đầu mỗi ca khám. Nếu đã trôi qua → tự động ẩn, không cho đặt lịch trong quá khứ.
Logic lọc giờ (Flutter)
if (selectedDate == today) { final now = TimeOfDay.now(); slots = slots.where((slot) { final parts = slot.startTime .split(':'); final slotTime = TimeOfDay( hour: int.parse(parts[0]), minute: int.parse(parts[1]), ); return slotTime.hour > now.hour || (slotTime.hour == now.hour && slotTime.minute > now.minute); }).toList(); }
